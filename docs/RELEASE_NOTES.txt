OVERVIEW
--------
Build 5040 is a targeted maintenance release that resolves every known
limitation from Build 5040. No new modules are added; the focus is on
removing hard requirements, adding automatic fallbacks, and ensuring the
tool runs correctly on every supported Windows build.

LIMITATIONS RESOLVED
--------------------

[FIX] ISO Creator -- oscdimg / Windows ADK dependency
  BEFORE: If oscdimg was not installed and the ADK auto-install failed, the
          ISO build aborted with no output.
  AFTER:  Three-tier fallback chain:
            1. oscdimg.exe (Windows ADK) -- fully bootable ISO
            2. wimlib-imagex             -- fully bootable ISO
            3. .NET System.IO.Compression.ZipFile -- always available,
               produces an archive; displays a clear warning that the result
               is non-bootable and ADK is recommended for bootable output.
          No build now fails silently due to a missing external tool.

[FIX] ISO App Embedding -- winget not available on target system
  BEFORE: Install-Apps.bat checked for winget and exited if absent, requiring
          the user to manually install App Installer first.
  AFTER:  Install-Apps.bat bootstraps winget automatically in two steps:
            1. Add-AppxPackage -RegisterByFamilyName
               Microsoft.DesktopAppInstaller_8wekyb3d8bbwe
            2. If still absent: downloads aka.ms/getwinget and installs via
               Add-AppxPackage
          Also runs "winget list --accept-source-agreements" silently before
          any install to pre-accept all source agreements.

[FIX] Chocolatey -- separate manual install required
  BEFORE: If winget failed to install an app and a Chocolatey package ID was
          available, the install was silently skipped when choco.exe was absent.
  AFTER:  When choco.exe is not found and an app has a Chocolatey ID, the
          install job automatically downloads and runs the official Chocolatey
          bootstrap script from community.chocolatey.org/install.ps1, then
          updates PATH and retries the installation. All output is written to
          the WinTooler log.

[FIX] Startup Manager -- TaskScheduler service dependency
  BEFORE: Disabling a startup item that was a scheduled task silently failed
          if the Task Scheduler service (Schedule) was stopped.
  AFTER:  Both Enable and Disable handlers now:
            1. Check the "Schedule" service status via Get-Service
            2. Call Start-Service if stopped, with 800ms settle delay
            3. Use schtasks.exe /Change /TN <name> /DISABLE|ENABLE as a
               CLI fallback for scheduled-task entries (works independently
               of the service COM layer)

[FIX] Segoe MDL2 Assets icons -- Windows 10 builds before 19041
  BEFORE: All sidebar nav icons, tool icons and status icons were invisible on
          Windows 10 versions older than 20H1 (build 19041).
  AFTER:  Two-layer fix:
            1. XAML FontFamily updated to include Segoe UI Symbol as third
               fallback: "Segoe MDL2 Assets, Segoe Fluent Icons, Segoe UI Symbol"
               Segoe UI Symbol is available since Windows 7 and covers the
               codepoints used by WinTooler.
            2. At window load time, if OSBuild < 19041, a VisualTreeHelper
               walk patches every TextBlock whose FontFamily contains
               "MDL2" or "Fluent" to use Segoe UI Symbol directly.
          Icons now render correctly on all supported Windows 10 builds.
